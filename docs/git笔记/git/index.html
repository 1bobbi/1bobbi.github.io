git

1.配置：

​ name git config --global user.name "bbb"

​ email git config --global user.email "fhhh_66@qq.com"

2.使用：

​   git status 查看当前仓库的状态

​   git init    初始化仓库

​   刚添加的文件处于未跟踪

​       未跟踪 ---> 暂存     git add  <filename>

​       暂存 ---> 入库（未修改） git commit -m 'xxx'

​       未修改 ---> 修改 修改代码,修改完之后重复上述步骤

​   git commit -a -m 'xxx'  提交所有未修改(未跟踪的不会提交)

​   git log     查看日志

3.常用命令

git restore <filename>  恢复文件

git restore --staged <filename> 取消暂存状态

git rm <filename>   删除  -f  强制

git mv from to  移动文件，重命名文件

4.分支

​ git在存储文件时，每一次代码提交都会创建一个与之对应的节点，git通过一个一个节点来记录代码状态，节点会构成一个树状结构，树状结构就意味着这个树会存在分支，默认情况下仓库只有一个分支，命名为master 

git branch  查看分支

git branch <name>   创建新分支

git branch -d <name>    删除分支

可以创建多个分支，修改代码时分支之间互不影响

git switch <name>   切换分支

git switch  -c <name>   创建并切换分支

在开发中，都是在自己的分支上编写代码，完成后再将自己的分支合并到主分支中

git merge <own-name>

在开发中，除了通过merge来合并分支以外，还可以通过变基( rebase )来完成分支的合并

我们通过merge合并分支时，在提交记录中会将所有的分支创建和分支合并的过程全部显示出来，这样当项目比较复杂，开发过程比较波折时，要进行反复创建，合并，删除，这样会使我们代码的提交记录变得极为混乱

原理：

​ 1.当我们发起变基时，git会首先找到两条分支的最近的共同祖先

​ 2.对比当前分支相对于祖先的历史提交，并且将他们提取出来存储到临时文件中

​ 3.将当前部分指向目标的基底

​ 4.以当前基底开始，重新执行历史操作

git rebase master

git switch master

git merge <own-name>

变基和merge对合并来说最终的结果是一样的，但是编辑会使得代码的提交记录更整洁，清晰

大部分情况下合并和变基是可以互换的，但是如果分支已经提交给了远程仓库，最好不要使用变基

5.远程仓库( remote )

开发我们需要一个远程仓库，远程仓库和本地仓库本质没有什么区别，不同点在于远程的仓库可以被多人同时访问使用，在实际工作中，git服务器通常由公司搭建内部使用或者购买一些公共的私有git服务器

目前常用的库：GitHub和Gitee

将本地库上传GitHub：

git remote add <remote name> <url>
git branch -M main #修改分支的名字为main
git push -u <remote name> <分支名字> #将代码上传到服务器上

将本地库上传Gitee：

git remote add <remote name> <url>
git push -u <remote name> <分支名字>

远程库操作的命令

git remote 列出当前关联的远程库
git remote add <远程库名> <url> 关联远程仓库
git remote remove <远程库名> 删除远程库
git push -u <远程库名> <分支名> 向远程库推送代码并和当前分支关联
git push <远程库名> <本地分支>:<远程分支>
git clone <url> 从远程库下载代码
git push 如果本地库的版本低于远程库，push默认是推不上去的
git fetch 要想推送成功，必须先确保本地库和远程库版本一致，fetch他会从远程仓库下载所有代码，但是他不会将代码和当前分支自动合并，所以使用fetch拉取代码后必须要手动对代码进行合并
git pull 从服务器上拉取代码并自动合并

推送代码之前要先从远程库中拉取最新的代码

6.标签

当头指针没有指向某个分支的头部时，这种状态叫做分离头指针（HEAD detached）,分离头指针的状态下也可以操作，但是这种操作不会出现在任何分支上，所以不要在分离头指针状态下进行操作

如果一定要进行操作，可以选择创建新分支后进行操作

git switch -c <分支名> <提交id>

可以为提交记录设置标签，设置标签以后，可以通过标签快速识别出不同节点

git tag <标签名> 
git tag <标签名> <提交id>
git push <远程仓库> <标签名>
git push <远程仓库> --tages
git tag -d <标签名> 删除标签
git push <远程仓库> --delete <标签名> 删除远程标签

7.gitignore

默认情况下，git会监视项目中的所有内容，但是有些内容，比如node_modules目录中的内容，我们不希望它被git管理，我们可以在项目目录中添加.gitignore文件，来设置那些需要git忽略的文件

8.github的静态页面

在github中可以将自己的静态页面直接部署到github，他会给我们提供一个地址使得我们页面变成一个真正的网站，可以供用户访问

要求：

​ 静态页面的分支必须叫做：gh-pages

​ 如果希望页面可以通过xxx.github.io访问，则需要将库的名字配置为xxx.github.io
