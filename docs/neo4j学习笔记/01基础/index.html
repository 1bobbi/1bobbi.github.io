01基础

CQL​

1.CQL基本语法

create

用来创建节点和关系

create （nodename：labelname{属性名：属性值})，string类的属性要加单引号或双引号

match

获取有关节点，关系和属性的数据

match (nodename:labelname) --- 会报错提示match要和其它语句配合使用

# 查询Dept下的内容

MATCH (dept:Dept) return dept

#查询Employee标签下 id=123，name="Lokesh"的节点

MATCH (p:Employee {id:123,name:"Lokesh"}) RETURN p

#查询Employee标签下name="Lokesh"的节点，使用（where命令）

MATCH (p:Employee) WHERE p.name = "Lokesh" RETURN p

return子句

检索节点和关联关系的属性

return nodename.属性名

上图报错是因为return语句不能单独使用

match+return的匹配和返回

match或return都不能单独使用，因此我们应该合并这两个命令从数据库检索数据

检索节点和关联关系的属性

match (nodename:labelname)

return nodename.属性名 / nodename

2.关系基础

neo4j关系被分为两种主要类型

  • 单向关系
  • 双向关系

neo4j使用类似的箭头来创建两个节点之间的关系

每个关系（->)包含两个节点，从节点->到节点

2.1没有属性的关系与现有节点

有节点无属性，创建两个节点之间的关系

match （node1-label-name:node1name),(node2-label-name:node2name)

create(node1-label-name)-[关系标签名：关系名{分配给新创建关系的属性}] ->（node2-label-name)

return 关系标签名

node1-label-name: from node 的名称

node2-label-name: to node 的名称

return是可选的，想立即看到结果可以使用

2.2现有节点的属性的关系

创建已有节点与属性之间的关系

match （node1-label-name:node1name),(node2-label-name:node2name)

create(node1-label-name)-[关系标签名：关系名{分配给新创建关系的属性}] ->（node2-label-name)

return 关系标签名

与2.1相同

2.3新节点无属性关系

create

(node1-label-name:node1name) -

[关系标签名：关系名{分配给新创建关系的属性}] ->

(node2-label-name:node2name)

return 关系标签名

2.4新节点与属性的关系

create

(node1-label-name:node1name{属性名：属性值}) -

[关系标签名：关系名{分配给新创建关系的属性}] ->

(node2-label-name:node2name{属性名：属性值})

return 关系标签名

2.5检索关系节点的详细信息

match (node1-label-name) - [关系标签名：关系名{分配给新创建关系的属性}] -> (node2-label-name)

return 关系标签名

3.create创建标签

3.1单个标签到节点

create （nodename：labelname)

3.2多个标签到节点

create (<node-name>:<label-name1>:<label-name2>.....:<label-name>)

3.3单个标签到关系

create (<node1-name>:<label1-name>)-
    [关系标签名：关系名{分配给新创建关系的属性}]
    ->(<node2-name>:<label2-name>)

where子句

where用来过滤match查询的结果

语法：

1.where condition

2.where condition boolean-operator condition

condition语法

property-name comparison-operator value

property-name：节点或关系的属性名称

comparison-operator：比较运算符

=：等于 < >：不等于 <：小于 >：大于 <= / >=：小于等于/大于等于

value：字面值，如数字文字，字符串文字等。

boolean-operator布尔运算符

AND OR NOT XOR

使用where子句创建关系

MATCH (<node1-label-name>:<node1-name>),(<node2-label-name>:<node2-name>) 
WHERE <condition>
CREATE (<node1-label-name>)-[<relationship-label-name>:<relationship-name>
       {<relationship-properties>}]->(<node2-label-name>)

delete

1.删除节点

DELETE <node-name-list>

node-name-list：删除的节点名称列表。

2.删除节点和关系

DELETE <node1-name>,<node2-name>,<relationship-name>

使用逗号（，）运算符来分隔节点名。

remove

1.删除节点或关系的标签

REMOVE <label-name-list>

label-name-list：标签列表

2.删除节点或关系的属性

REMOVE property-name-list

property-name-list：属性列表

Neo4j CQL DELETE和REMOVE命令之间的主要区别 -

  • DELETE操作用于删除节点和关联关系。
  • REMOVE操作用于删除标签和属性。

Neo4j CQL DELETE和REMOVE命令之间的相似性 -

  • 这两个命令不应单独使用。
  • 两个命令都应该与MATCH命令一起使用。

set子句

  • 向现有节点或关系添加新属性
  • 添加或更新属性值
SET  <property-name-list>

order by 排序

对MATCH查询返回的结果进行排序

ORDER BY  <property-name-list>  [DESC]

property-name-list：属性列表

union合并

1.union子句

<MATCH Command1>
   UNION
<MATCH Command2>

将两组结果中的公共行组合并返回到一组结果中。 它不从两个节点返回重复的行。

结果列类型和来自两组结果的名称必须匹配，这意味着列名称应该相同，列的数据类型应该相同。

如果这两个查询不返回相同的列名和数据类型，那么它抛出一个错误。

2.union all 子句

<MATCH Command1>
UNION ALL
<MATCH Command2>

结合并返回两个结果集的所有行成一个单一的结果集。它还返回由两个节点重复行。

结果列类型，并从两个结果集的名字必须匹配，这意味着列名称应该是相同的，列的数据类型应该是相同的。

如果这两个查询不返回相同的列名和数据类型，那么它抛出一个错误。

limit和skip子句

“LIMIT”子句来过滤或限制查询返回的行数，修剪CQL查询结果集底部的结果

LIMIT <number>

“SKIP”子句来过滤或限制查询返回的行数，修整了CQL查询结果集顶部的结果---跳过几个结果

SKIP <number>

合并

Neo4j使用CQL MERGE命令 -

  • 创建节点，关系和属性
  • 为从数据库检索数据

MERGE命令是CREATE命令和MATCH命令的组合。

MERGE = CREATE + MATCH

MERGE命令在图中搜索给定模式，如果存在，则返回结果

如果它不存在于图中，则它创建新的节点/关系并返回结果。

MERGE (<node-name>:<label-name>
{
   <Property1-name>:<Pro<rty1-Value>
   .....
   <Propertyn-name>:<Propertyn-Value>
})

null值

Neo4j CQL将空值视为对节点或关系的属性的缺失值或未定义值。

当我们创建一个具有现有节点标签名称但未指定其属性值的节点时，它将创建一个具有NULL属性值的新节点。

in操作符

IN[<Collection-of-values>]

Collection-of-values：由逗号运算符分隔的值的集合。

CQL函数​

字符串函数

UPPER：将所有字母更改为大写字母。

UPPER (<input-string>)

input-string 可以是来自Neo4J数据库的节点或关系的属性名称。

LOWER：将所有字母改为小写字母。

LOWER (<input-string>)

SUBSTRING：获取给定String的子字符串

SUBSTRING(<input-string>,<startIndex> ,<endIndex>)

在Neo4J CQL中，如果一个字符串包含n个字母，则它的长度为n，索引从0开始，到n-1结束。

startIndex 是SUBSTRING函数的索引值。

endIndex 是可选的。 如果我们省略它，那么它返回给定字符串的子串从startIndex到字符串的结尾。

REPLACE：替换一个字符串的子字符串。

aggregation聚合

COUNT：返回由MATCH命令返回的行数

COUNT(<value>)

value 可以是*，节点或关系标签名称或属性名称

MAX: 从MATCH命令返回的一组行返回最大值

MAX(<property-name> )
用一组行和节点或关系的<property-name>作为输入，并从给定行的give <property-name>列中查找最大值

<property-name>应该是节点或关系的名称。

MIN: 返回由MATCH命令返回的一组行的最小值

MIN(<property-name> )

SUM: 返回由MATCH命令返回的所有行的求和值

SUM(<property-name> )

AVG: 返回由MATCH命令返回的所有行的平均值

AVG(<property-name> )

关系函数

STARTNODE：用于知道关系的开始节点

ENDNODE：用于知道关系的结束节点

STARTNODE (<relationship-label-name>)
ENDNODE (<relationship-label-name>)
<relationship-label-name>可以是来自Neo4j数据库的节点或关系的属性名称。

ID：用于知道关系的ID

TYPE：用于知道字符串表示中的一个关系的TYPE

其它​

索引

Neo4j SQL支持节点或关系属性上的索引，以提高应用程序的性能。

我们可以为具有相同标签名称的所有节点的属性创建索引。

我们可以在MATCH或WHERE或IN运算符上使用这些索引列来改进CQL Command的执行。

创建索引

CREATE INDEX ON :<label_name> (<property_name>)

冒号（:)运算符用于引用节点或关系标签名称。

上述语法描述它在节点或关系的<label_name>的<property_name>上创建一个新索引

删除索引

DROP INDEX ON :<label_name> (<property_name>)

uniqe约束

  • 避免重复记录。
  • 强制执行数据完整性规则。

创建唯一的约束

CREATE CONSTRAINT ON (<label_name>)
ASSERT <property_name> IS UNIQUE

删除约束

DROP CONSTRAINT ON (<label_name>)
ASSERT <property_name> IS UNIQUE
