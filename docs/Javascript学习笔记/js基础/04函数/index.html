04函数

函数---是一个对象，可以封装一些功能

创建函数对象

可以将要封装的代码以字符串的形式传递给函数，代码在调用函数时执行

 var fun = new Function();
 var fun = new Function("console.log('hello');");
 fun();

使用函数声明创建函数

语法：function 函数名([形参1，形参2……]){

​ }

function fun2(){
    document.write("world");
}

使用函数表达式创建函数

var fun3 = function(){
     document.write("world");
 };

函数的参数：调用函数时解析器不会检查实参的类型，有可能会接收到非法参数，也不会检查实参的数量，多输实参不会赋值，实参数量小于形参，没有对应的形参将是undefined

返回值：return无值就是undefined，可以返回任意类型的值，返回值可以是任意类型，如对象，函数

立即执行函数：

function(){
     alert("hhh");
 }//匿名函数，会报错，浏览器会认为{}之间是代码块
(function(){
      alert("hhh");
 })();//加括号，变成整体，立即执行

作用域：一个变量的作用范围，在js中有两种作用域（全局和函数（局部））

全局：直接编写在script标签中的，

​ 其中有一个全局对象：window，它代表的是一个浏览器窗口

​ 在全局作用域中，我们创建的变量都会作为window的属性保存，创建的函数都会作为window的方法

变量的声明提前：使用var关键字声明的变量会在所有代码执行前被声明

函数的声明提前：

使用函数声明创建的函数，会在所有代码执行之前就被创建，所以我们可以在函数声明前调用函数

fun();
 function fun(){
      console.log("ghj");
 }

使用函数表达式创建的则不能能

a();
var a = function(){
    console.log("shj");
}//会报错

全局作用域中的变量都是全局变量

函数：在函数作用域中可以访问全局变量，反之不行

​ 在函数作用域中操作变量时，先在自身作用域寻找，在函数访问全局变量可使用window对象

var c = 10;
function b(){
    console.log(c);
    c = 20;
}
 b();//10

​ 在函数作用域中也有声明提前特性

var c = 10;
function b(){
   console.log(c);
   var c = 20;
}
b();//undefined

​ 在函数中不使用var声明的变量都会成为全局变量

var c = 10;
function b(){
    console.log(c);
    c = 20;
}
console.log(c);//20

​ 定义形参就相当于在函数作用域中声明了变量

var e = 10;
function f(e){
    console.log(e);
    e = 10;
}
f();//undefined

this：解析器调用函数时每次都会像函数内部传递一个隐含参数，这个隐含参数就是this

​ this指向的是对象，我们称为函数执行的上下文对象

​ 根据函数的调用方式不同，this会指向不同的对象：

​ 1.以函数形式调用时，this指window

​ 2.以方法形式调用时，this就是调用方法的那个对象

        function f1(){
            //console.log(a,b);
            console.log(this);
        }
        
        var a = {
            name : "xiaoming",
            f:f1
        };
        //console.log(a.f == f1);true
        a.f();//object
        f1();//window

构造函数：普通的函数，不同的是构造函数习惯上首字母大写，调用方式和普通函数不同

普通函数直接调用，构造函数需要new关键字

创建一个构造函数

        function Persion(){

        }
        var per = Persion();
        console.log(per);//undefined
        var per = new Persion();
        console.log(per);//object

执行流程：

1.立即创建新对象

2.将新建对象设置为函数中的this，在构造函数中使用this引用新建对象

3.执行函数中的代码

4.将新建对象返回

使用同一个构造函数创建的对象，称为一类对象，也将构造函数成为一个类

通过类创建一个对象，就是该类的实例

使用instanceof可以检查一个对象是否是一个类的实例，所有对象都是object的实例

 console.log(per instanceof Persion);

原型prototype：我们所创建的每一个函数，解析器都会添加一个属性prototype

​ 这个属性对应着一个对象，这个对象就是原型对象

如果函数作为普通函数调用prototype没有任何作用

构造函数调用时，他所创建的对象中都会有一个隐含的属性指向该构造函数的原型对象，可通过"___proto"+ 下划线来访问该属性

原型对象相当于一个公共区域，所有同一个类的实例都可以访问到这个原型对象中

function Persion(){
}
var per = new Persion();
console.log(Persion.prototype == per_proto_);//true

可以使用对象的hasOwnProperty（）来检查对象自身是否含有该属性

console.log(per.hasOwnProperty("name"));

原型对象也是对象，他也有原型

使用对象的属性或方法时，会先在自身寻找，没有去原型中找，没有去原型的原型中寻找，直到找到object的原型，如果依然没有，就返回undefined

toString：当我们直接在页面中打印对象时，输出的实际上是对象的toString（）方法的返回值

如果不希望在输出对象时输出[object object],可为对象添加一个toString（）方法

垃圾回收（GC）：程序运行过程中会产生垃圾，这些垃圾积攒过多以后，回到致程序运行速度过慢

我们需要一个垃圾回收机制处理

当一个对象没有任何变量或属性对他进行引用，此时我们无法操作该对象，他就成为了垃圾

清理：在js中，有自动垃圾回收机制

函数的方法---需要通过函数对象来调用

call（） apply（）

当函数调用这两个方法时会调用函数执行

语法：函数.方法

在调用这两个方法时可以将一个对象指定为第一个参数，此时这个参数将会成为函数执行时的this 

        function fu(){
            console.log(this.name);
        }

        var t1 = {
            name:"t1",
            sayName:fu
        }
        fu.call(t1);

call()方法可以将实参在对象之后依次传递

apply（）需要将实参封装到数组里统一传递

        function fu(){
            //console.log(this.name);
            console(a);
            console(b);
        }

        var t1 = {
            name:"t1",
            sayName:fu
        }
        fu.call(t1,2,3);
        fu.apply(t1,[2,3]);

arguments：在调用函数时，浏览器每次都会传递两个参数，一个是this，一个是封装实参的对象arguments

arguments是一个类数组对象，它也可以通过索引操作数据，也可以获取长度，在调用函数时，我们传递的实参都会在arguments中保存，即使不定义形参也可以用

        function f2(){
            console.log(arguments.length);
        }
        f2("hh",1);

他里边有个属性:callee，它对应一个函数对象，就是当前正在指向函数的对象
