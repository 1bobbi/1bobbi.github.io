函数高级

一，

原型

1.函数的prototype属性

每个函数都有prototype属性，它默认指向一个object空对象（即称为：原型对象）

空对象：没有我们的属性

function fun(){

}
console.log(fun.prototype);//空对象
fun.prototype.test = function(){//添加属性
    console.log("hello");
}

原型对象中都有一个属性constructor，它指向函数对象

console.log(fun.prototype.constructor === fun);//true

2.给原型对象添加属性（一般都是方法）

作用：函数的所有实例对象自动拥有原型中的属性（方法）

var f = new fun();
f.test();

3.显式原型与隐式原型

3.1每个函数都有一个prototype，即显式原型（属性）

3.2每个实例对象都有一个“ ” + proto+ “ ” ，可称为隐式原型（属性）

3.3对象的隐式原型的值为其对应构造函数的显示原型的值

function Fn(){//内部语句：Fn.prototype = {}
}
console.log(Fn.prototype);//显式，默认指向一个空的object对象
var fn = new Fn();//内部语句：this.__proto__ = Fn.prototype
console.log(fn.__proto__);//隐式
console.log(fn.__proto__ === Fn.prototype);//true

总结：prototype属性在定义函数时自动添加，_proto__在创建对象时自动添加

程序员能直接操作显式原型，但不能直接操作隐式（ES6之前）

原型链

访问一个对象的属性时：

​ 先在自身属性中查找，找到返回

​ 如果没有，再沿着隐式原型这条链向上找，找到返回

​ 如果最终没有找到，返回undefined

别名：隐式原型链

作用：查找对象的属性（方法）

所有函数的隐式原型都是相同的

2.补充：

2.1函数的显式原型指向的对象默认是空的object实例对象（但object不满足，它为空）

console.log(Fn.prototype instsanceof object)//true
console.log(object.prototype instsanceof object)//false
console.log(Function.prototype instsanceof object)//true

2.2所有函数都是Function的实例（包括Function）

console.log(Function.__proto__ === Function.prototype)//true

object的实例对象是原型链的尽头

console.log(object.prototype.__proto__)//null

原型链—属性问题

1.读取对象属性值时，会自动到原型链中查找

2.设置对象属性值时，不会查找原型链1，如果当前对象中没有此属性，直接添加属性并设置其值

3.方法一般定义在原型中，属性一般通过构造函数定义在对象本身上

作用域与作用域链

1.理解

作用域就是一块“地盘”，一个代码段所在的区域，它是静态的（相当于上下文对象），在编写代码时就确定了

作用域链是多个上下级关系的作用域形成的链，方向从下往上（由内到外),查找变量时沿作用域链，现在当前作用域找，有返回，没有向上一级

作用域：

2.分类

全局

函数

没有块作用域（ES6有了）：在{ }里声明的变量是可见的

个数：n（定义了n个函数）+1

3.作用

隔离变量，不同作用域下同名变量不会有冲突

作用域与执行上下文

1.区别1

全局作用域之外，每个函数都会创建自己的作用域，作用域在函数时就已经确定了，而不是在函数调用时

全局执行上下文环境是在全局作用域确定之后，js代码马上执行之前创建

函数执行上下文是在调用函数时，函数体代码执行之前创建

2.区别2

作用域是静态的，只要函数定义好了就一直存在，且不会再变化

执行上下文是动态的，调用函数时创建，函数调用结束时就会自动释放

3.联系

执行上下文是从属于所在的作用域

全局上下文环境==》全局作用域

函数上下文环境==》对应的函数使用域

1.如何产生闭包？

当一个嵌套的内部（子）函数引用了嵌套的外部（父）函数的变量（函数）时，就产生了闭包

function fn1(){
    var a = 2
    var b = 'abc'
    function fn2(){ 
        console.log(a)
    }
}
fn1()

2.闭包到底是什么？

理解一：闭包是嵌套的内部函数

理解二：包含被引用变量（函数）的对象

3.产生闭包的条件？

函数嵌套，内部函数引用了外部函数的数据（变量/函数），执行外部函数--》执行内部函数定义--》产生闭包，也可以执行内部函数

常见的闭包

1.将函数作为另一个函数的返回值

function fn1(){
    var a = 2
    function fn2(){
        a++
        console.log(a)
    }
    return fn2
}
var f = fn1()
f()// 3
f()// 4
//只产生了一个闭包
//再创建一个
fn1()

2.将函数作为实参传递给另一个函数使用

function showDelay(msg,time){
    setTimeout(function(){
        alert(msg)
    },time)
}
showDelay('atguigu',2000)

闭包的作用

1.使函数内部的局部变量在函数执行完后，仍然存活在内存中（延长局部变量的生命周期）

2.让函数外部可以操作（读写）到函数内部的数据（变量/函数）

问题：

function fn1(){
    var a = 2
    function fn2(){
        a++
        console.log(a)
    }
    return fn2
}
var f = fn1()

函数执行完后，函数内部声明一般不存在，存在于闭包中的存在，如：fn2本身不存在，被释放了，但函数对象没有成为垃圾对象，因为f在引用函数对象

在函数外部不能访问内部的局部变量，但我们通过闭包让外部操作它

闭包的声明周期

1.产生：在嵌套内部函数定义执行完就产生了（不是在调用）

2.死亡：嵌套内部函数成为垃圾对象

function fn1(){
    var a = 2   //此时闭包已经产生，因为函数提升，内部函数对象已经创建了
    function fn2(){
        a++
        console.log(a)
    }
    return fn2
}
var f = fn1()
f = null //闭包死亡（包含闭包的函数对象成为了垃圾对象）

闭包的应用

自定义js模块：

具有特定功能的js文件

将所有的数据和功能都封装在一个函数内部（私有的）

对象的属性不是私有的，函数里是

只向外暴露一个包含n个方法的对象或函数

模块的使用者，只需要通过模块暴露的对象调用方法来实现对应的功能

mym2.dothing1()
mym2.dothing2()

闭包的缺点及解决

缺点：函数执行完后，函数内的局部变量没有释放，占用内存的时间会变长；容易造成内存泄漏

function fn(){
    var arr = new Array[10000]
    function fn2(){
        console.log(arr.length)
    }
    return fn2
}
var f = fn1()

解决：能不用闭包就不用，及时释放

f = null // 让内部函数称为垃圾对象---》回收闭包

内存溢出

一种程序运行出现的错误

当程序运行需要的内存超过剩余内存时，就抛出内存溢出错误

var obj = {}
for(var i = 0;i < 10000 ; i ++){
    obj[i] = new Array(10000)
}

内存泄漏

占用的内存没有及时释放

内存泄漏积累多了就容易导致内存溢出

常见的内存泄漏：

意外的全局变量

没有及时清理的计时器或回调函数
